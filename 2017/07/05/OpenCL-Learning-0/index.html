OpenCL学习(0)

OpenCL 工作的流程：

  • 寻找所有可用的Platform（Intel，AMD，NVIDIA等），一个Platform对应一个OpenCL编译器
  • 选择一个Platform，在这个Platform上建立一个Context（直译为上下文，Platform与Kernel之间交换数据的中间层？）
  • 在Platform上建立一个或几个可用的Device
  • 建立一个Program对象，读入Kernel程序
  • 在指定的Device上编译指定的Program里的指定的Kernel程序
  • 为Kernel程序创建OpenCL内存对象（一维Buffer，二维及以上有Image，Texture等等）
  • 把C++程序中的数据映射到OpenCL内存对象

要执行某个Kernel程序时要做的：

  • 设置Kernel程序的参数（用OpenCL内存对象的内容）

  • 把Kernel程序装入Command Queue（每一个Kernel程序使用的内存空间互不干扰）

  • OpenCL把设备的物理多核（等价的）划分成虚拟的n核，同时执行n个kernel、

这一步随时做都可以：

  • 把执行结果（表现为Kernel程序里面的某个参数，存在某个OpenCL内存对象里面）读回C++程序中
